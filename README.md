# Gerenciador de atividades

## Objetivo do projeto

O sistema de gerenciamento de atividades organiza e prioriza tarefas utilizando estruturas de dados, proporcionando ao usuário uma forma eficiente de controlar suas atividades.

## justificativa da escolha do tema

Devido à falta de tempo e à dificuldade de organização nos estudos, o desenvolvimento de um sistema para gerenciar atividades mostrou-se uma escolha viável e relevante como primeiro projeto com Python. 
Além de refletir uma necessidade real do dia a dia, o tema permitiu aplicar conceitos fundamentais de estruturas de dados de forma prática e significativa.

## Estruturas utilizadas.

  1. Fila: a fila foi utilizada para armazenar as atividades que eram descritas para o usuário. 
  2. Heap: o heap armazena de forma prioritária as atividades que mereciam mais atenção descrita pelo usuário.
  3. Lista encadeada: conforme as atividades eram concluídas, a lista encadeada servia mais como um histórico que guardava as informações das atividades já feitas.


## Desafios enfrentados e soluções encontradas.

#### Desafios:

  1. O primeiro contato com as estruturas apresentadas.
  2. A utilização de algumas bibliotecas, como deque e heapq.
  3. A junção de todas as estruturas.
     
#### Soluções:

  1. A facilidade de encontrar informações com documentações.
  2. A utilização de algumas estruturas facilitou a compreensão de outras.		
  3. Instruções para executar o projeto.

## Instruções para executar o projeto.

  1. Utilização de algum Editor de texto(Visual Studio Code,PyCharm)
  2. Executar o arquivo index.py no prompt de comando
